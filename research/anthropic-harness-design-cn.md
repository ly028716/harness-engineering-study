# Anthropic 三智能体 Harness 设计详解

> 基于多个来源整理的 Anthropic Harness 设计核心内容

## 概述

Anthropic 提出了一个三智能体（Three-Agent）Harness 框架，专门用于支持长时间运行的全栈 AI 开发任务。这个框架通过将代码生成与评估分离，显著提高了 AI Agent 在复杂任务中的可靠性和输出质量。

## 核心设计理念

### 关键问题：任务漂移（Task Drift）

**什么是任务漂移？**
- AI Agent 在长时间运行任务时，逐渐偏离原始目标
- 随着上下文积累，Agent 可能"忘记"初始需求
- 中间错误会累积，导致最终结果偏离预期

**Anthropic 的解决方案**：
- 分离生成和评估职责
- 使用专门的 Agent 进行质量把控
- 建立清晰的反馈循环

## 三智能体架构

### Agent 1：Generator（生成器）

**职责**：
- 理解任务需求
- 生成代码实现
- 创建初始解决方案
- 快速迭代和试错

**特点**：
- 专注于创造性工作
- 不负责质量评估
- 可以大胆尝试不同方案
- 输出需要经过验证

**工作流程**：
```
接收任务 → 分析需求 → 生成代码 → 提交给 Evaluator
```

### Agent 2：Evaluator（评估器）

**职责**：
- 评估生成的代码质量
- 运行测试和验证
- 识别问题和缺陷
- 提供详细反馈

**特点**：
- 专注于质量把控
- 客观评估输出
- 不参与代码生成
- 提供改进建议

**评估维度**：
- 功能正确性
- 代码质量
- 性能表现
- 安全性
- 可维护性

**工作流程**：
```
接收代码 → 运行测试 → 分析结果 → 生成反馈报告 → 发送给 Coordinator
```

### Agent 3：Coordinator（协调器）

**职责**：
- 管理整体工作流程
- 协调 Generator 和 Evaluator
- 决策是否需要重新生成
- 跟踪任务进度
- 处理异常情况

**特点**：
- 全局视角
- 决策权威
- 状态管理
- 人机接口

**决策逻辑**：
```
if 评估通过:
    标记任务完成
    进入下一个任务
else if 可以改进:
    将反馈发送给 Generator
    请求重新生成
else if 超过重试次数:
    请求人工介入
    暂停任务
```

## 工作流程

### 完整循环

```
1. Coordinator 接收任务
   ↓
2. Coordinator 将任务分配给 Generator
   ↓
3. Generator 生成代码
   ↓
4. Generator 将代码提交给 Evaluator
   ↓
5. Evaluator 运行测试和验证
   ↓
6. Evaluator 生成评估报告
   ↓
7. Coordinator 分析评估结果
   ↓
8. 决策分支：
   - 通过 → 完成任务
   - 需改进 → 返回步骤 2（带反馈）
   - 失败 → 人工介入
```

### 多小时会话支持

**设计特点**：
- 支持长时间运行（数小时）
- 保持上下文连贯性
- 定期检查点保存
- 可恢复的状态管理

**实现机制**：
- 持久化任务状态
- 增量式进度保存
- 上下文压缩和管理
- 定期同步和验证

## 核心设计原则

### 1. 职责分离（Separation of Concerns）

**原则**：
- 生成和评估必须分离
- 每个 Agent 专注于自己的职责
- 避免角色混淆

**好处**：
- 提高可靠性
- 减少偏见
- 更容易调试
- 可以独立优化

### 2. 反馈驱动（Feedback-Driven）

**原则**：
- 基于评估结果进行改进
- 提供具体、可操作的反馈
- 持续迭代直到满足要求

**实现**：
- 详细的评估报告
- 具体的改进建议
- 可追踪的问题列表
- 进度可视化

### 3. 状态管理（State Management）

**原则**：
- 清晰的状态定义
- 可恢复的检查点
- 状态转换的可追踪性

**实现**：
- 持久化关键状态
- 版本控制
- 状态快照
- 回滚机制

### 4. 人机协作（Human-in-the-Loop）

**原则**：
- 人类保留最终决策权
- 关键节点需要人工确认
- 提供清晰的介入接口

**实现**：
- 人工审查点
- 异常处理机制
- 进度报告
- 交互式调整

## 技术实现要点

### 上下文管理

**挑战**：
- 长时间运行导致上下文膨胀
- 需要保持关键信息

**策略**：
- 上下文压缩
- 关键信息提取
- 分层上下文管理
- 定期刷新

### 错误处理

**策略**：
- 分类错误类型
- 自动重试机制
- 降级策略
- 人工介入触发

**实现**：
```python
class ErrorHandler:
    def handle_error(self, error, context):
        if error.is_recoverable():
            return self.retry_with_adjustment(context)
        elif error.is_critical():
            return self.request_human_intervention(error)
        else:
            return self.apply_fallback_strategy(context)
```

### 质量门禁

**层次**：
1. 语法检查（立即）
2. 单元测试（快速）
3. 集成测试（中速）
4. 端到端测试（慢速）
5. 人工审查（按需）

**实现**：
```python
class QualityGate:
    def validate(self, code):
        results = []
        results.append(self.syntax_check(code))
        if results[-1].passed:
            results.append(self.unit_tests(code))
        if results[-1].passed:
            results.append(self.integration_tests(code))
        return QualityReport(results)
```

## 适用场景

### 最适合的场景

1. **全栈应用开发**
   - 前后端协同
   - 多个组件集成
   - 需要长时间开发

2. **复杂功能实现**
   - 需要多次迭代
   - 质量要求高
   - 有明确的验证标准

3. **原型到产品**
   - 从概念到实现

   - 快速完成
   - 不需要复杂验证

2. **探索性编程**
   - 需求不明确
   - 频繁变更方向
   - 创造性探索

Anthropic 的三智能体 Harness 设计提供了一个可靠的框架，用于支持长时间运行的复杂 AI 开发任务。通过将生成、评估和协调分离，这个框架有效地解决了任务漂移问题，提高了输出质量和可靠性。

**核心价值**：
- 职责分离提高可靠性
- 专门的评估保证质量

- 支持长时间运行任务
